To use Terrateam with Azure, you need to configure authentication and authorization between Terrateam and your Azure account. Choose the setup method that works best for your needs.

## Setup Options

<CardGrid>
  <LinkCard
    title="Static Credentials"
    description="Easiest to get started - Set up an Azure service principal and store credentials in GitHub. Perfect for quick testing, simple setups, and getting started with Terrateam."
    href="/integrations/cloud-providers/azure/static-credentials"
  />
  <LinkCard
    title="OpenID Connect (OIDC)"
    description="Recommended for production - Secure, keyless authentication using GitHub's OIDC provider with Entra ID federated credentials. No secrets to manage or rotate."
    href="/integrations/cloud-providers/azure/oidc-setup"
  />
</CardGrid>

## What's Next?

Once you've configured Azure authentication, you'll be able to:

- Run `terrateam plan` to preview infrastructure changes
- Run `terrateam apply` to deploy infrastructure changes
- Use all of Terrateam's advanced features with your Azure resources

:::tip[Need help choosing?]
If you're just getting started with Terrateam, we recommend beginning with [static credentials](/integrations/cloud-providers/azure/static-credentials) as it's the quickest way to get up and running. You can always switch to OIDC later for enhanced security.
:::
