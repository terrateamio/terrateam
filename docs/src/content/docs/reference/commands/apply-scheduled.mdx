---
title: terrateam apply scheduled
description: Schedule a Terraform apply to run at a specific future time
---

Schedule an apply to execute automatically at a specified time. This is useful for planning infrastructure changes during maintenance windows or off-peak hours.

## Usage

```
terrateam apply [tag-query] at <ISO-8601-timestamp>
```

The timestamp must be in UTC and end with `Z`.

## Examples

### Schedule an Apply

```bash
# Schedule apply for a specific date and time
terrateam apply at 2026-03-24T14:00:00Z

# Schedule with a tag query
terrateam apply dir:production at 2026-03-24T02:00:00Z

# Schedule a specific workspace
terrateam apply workspace:prod at 2026-04-01T06:00:00Z

# Schedule with a complex tag query
terrateam apply dir:us-east-1 and production at 2026-03-25T22:00:00Z
```

### Cancel Scheduled Applies

```
terrateam apply-cancel
```

This cancels all pending scheduled applies on the current pull request.

## Confirmation

When a scheduled apply is created, Terrateam posts a confirmation comment on the pull request:

> Apply scheduled for **2026-03-24T14:00:00Z** UTC. To cancel, comment `terrateam apply-cancel`.

## Time Format

Only ISO 8601 timestamps in UTC are accepted. The timestamp must end with `Z`.

**Valid:**
- `2026-03-24T14:00:00Z`
- `2026-12-31T23:59:59Z`

**Invalid:**
- `2026-03-24T14:00:00` (missing `Z`)
- `2026-03-24T14:00:00+05:00` (not UTC)
- `14:00` (time only)

If an invalid timestamp is provided, Terrateam will post an error comment explaining the correct format.

## Behavior

- The server polls for due scheduled applies every 60 seconds.
- When the scheduled time arrives, the apply is triggered as if a user had commented `terrateam apply`.
- All standard apply requirements (approvals, checks, etc.) still apply at execution time.
- If the pull request is closed or merged before the scheduled time, the scheduled apply is not executed.
- Multiple scheduled applies can exist on the same pull request.
- `terrateam apply-cancel` cancels all pending scheduled applies on the pull request.

## Tag Query Support

Supports [tag queries](/reference/configuration/tag-queries) to filter which dirspaces to apply:

- `dir:<path>` - Target specific directory
- `workspace:<name>` - Target specific workspace
- Tags can be referenced directly (e.g., `production`, `staging`)
- Logical operators: `and`, `or`, `not`
